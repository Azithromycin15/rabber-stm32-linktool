# Release Notes - rabber-stm32-linktool v1.0.1

## Overview

This release improves the firmware flashing workflow, adds automatic MCU reset and status verification after flashing, and provides ready-to-use release binaries for both Linux and Windows x86_64.

## What changed

- Enhanced `flash` command behavior:
  - Automatically handles ELF and HEX files
  - Prompts for confirmation before flashing
  - Writes firmware to `0x08000000`
  - Automatically resets the MCU after successful flash
  - Verifies MCU startup status via SWD after reset
- Updated help output to describe the new auto-reset flashing workflow
- Fixed cross-platform compatibility in `src/utils.rs`
- Added release binaries for:
  - `release/rabber-stm32-linktool-1.0.1-linux`
  - `release/rabber-stm32-linktool-1.0.1-win64.exe`

## Release artifacts

- Linux release executable: `release/rabber-stm32-linktool-1.0.1-linux`
- Windows x64 release executable: `release/rabber-stm32-linktool-1.0.1-win64.exe`

## Notes

- Version bumped to `1.0.1` in `Cargo.toml`
- Tag created: `v1.0.1`
- Code pushed to both GitHub and Gitee

## Usage

Run the tool in interactive mode and use:

- `flash <file>` to flash ELF or HEX firmware and automatically reset the MCU
- `info` to query MCU information
- `reset` to manually reset the MCU
- `elf2hex <elf> <hex>` to convert ELF to HEX
